# md2pdf — Project Structure

> **Read this file first.** It maps every directory and file in the repository.
> Written for AI agents and new contributors who need to understand the project quickly.

---

## What This Project Is

`@amitdevx/md2pdf` is a TypeScript npm package that converts Markdown files to
production-quality PDFs using a unified/remark/rehype pipeline and Playwright (headless Chromium).

**Package name:** `@amitdevx/md2pdf`  
**Current version:** see `package.json`  
**Node.js requirement:** `>= 18`  
**Entry points:** `dist/index.js` (ESM), `dist/index.cjs` (CJS), `dist/cli/index.js` (CLI binary)

## `src/` — Source Code

See [`src/README.md`](./src/README.md) for the full module map.

```
src/
├── index.ts           ← Public API: exports convert(), defineConfig(), all types
├── cli/index.ts       ← CLI binary (bin: md2pdf) — thin wrapper around convert()
├── core/index.ts      ← convert() implementation — orchestrates the pipeline
├── parser/index.ts    ← Markdown → HTML via unified (remark + rehype)
├── renderer/index.ts  ← HTML template assembly (CSS injection, fonts, slots)
├── pdf/index.ts       ← Playwright PDF generation wrapper
├── pdf/metadata.ts    ← PDF-lib metadata injection via frontmatter
├── plugins/toc.ts     ← Rehype plugin for Table of Contents generation
├── plugins/           ← Future plugins (obsidian, custom blockquotes)
├── themes/            ← Theme CSS + metadata (default/, github/, obsidian-light/, obsidian-dark/)
├── config/            ← Config file loading, merging, validation
├── assets/            ← Static CSS (base.css, print.css, typography.css)
├── constants/         ← Shared constants
├── types/index.ts     ← All public TypeScript types
└── utils/             ← Shared utilities
```

**Pipeline:**
```
.md file → src/parser/ → src/renderer/ → src/pdf/ → .pdf file
```

## Build System

| Command | What It Does |
|---------|-------------|
| `npm run build` | `tsup` compiles `src/` → `dist/` (ESM + CJS + declarations) |

| `npm test` | Vitest: unit + E2E tests |
| `npm run lint` | ESLint |
| `npm run typecheck` | `tsc --noEmit` |
| `npm run clean` | Delete `dist/` |

## What Gets Published to npm

Only the `files` field in `package.json` is published:
```json
"files": ["dist", "README.md", "LICENSE", "CHANGELOG.md"]
```

**Never published:** `src/`, `tests/`, `docs/`, `examples/`, `scripts/`, `phase/`

## Key Design Principles

1. **CLI is a wrapper** — `convert()` API is first-class; CLI calls `convert()` with no extra logic
2. **Plugin pipeline from day one** — internal plugin architecture in v0.1.0; public API in v0.7.0
3. **Docs as you go** — every feature adds its doc to `docs/` at ship time, never deferred
4. **Config before themes** — `md2pdf.config.ts` (v0.5.0) makes themes (v0.6.0) feel cohesive
5. **Mermaid early** — flagship feature at v0.2.0, not buried behind unrelated work
